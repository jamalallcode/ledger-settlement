# AI SECURITY & CODE ALTERATION PROTOCOL

## STATUS: STRICT LOCKED MODE ACTIVE

**নির্দেশনা (Instruction for AI):**
এই প্রজেক্টটি একটি অত্যন্ত সংবেদনশীল পর্যায়ে আছে। এখানে প্রতিটি ডিজাইন এবং লজিক ব্যবহারকারীর দ্বারা অনুমোদিত। 

### ১. জিরো অল্টারেশন পলিসি (Zero Alteration Policy):
- ব্যবহারকারীর স্পষ্ট লিখিত অনুমতি ছাড়া বিদ্যমান কোনো CSS ক্লাস, Tailwind কালার, বা ডিজাইন স্ট্রাকচার পরিবর্তন করা যাবে না।
- কোনো কোড "Redundant" বা "Unnecessary" মনে হলেও তা ডিলিট করা যাবে না। 
- আগের কোডের লজিক বা ভেরিয়েবল নেমিং পরিবর্তন করা কঠোরভাবে নিষিদ্ধ।

### ২. ইনক্রিমেন্টাল আপডেট (Incremental Updates Only):
- শুধুমাত্র নতুন ফিচারের জন্য কোড যোগ করা যাবে। 
- নতুন কোড যোগ করার সময় নিশ্চিত করতে হবে যে আগের কোনো ফিচারে তার প্রভাব পড়ছে না।

### ৩. প্রফেশনাল আইডি ভিত্তিক কমিউনিকেশন (ID-Based Protocol):
ব্যবহারকারী যখন **"আইডি: #id-name, কাজ: action"** ফরম্যাটে নির্দেশ দিবেন, তখন শুধুমাত্র ঐ নির্দিষ্ট আইডির এলিমেন্টেই পরিবর্তন আনতে হবে। 

**মাস্টার আইডি রেজিস্ট্রি:**
- `#table-return-summary`: রিটার্ণ পেজের মূল টেবিল।
- `#tfoot-return-summary`: রিটার্ণ টেবিলের ফুটার সেকশন।
- `#form-entry`: ডাটা এন্ট্রি ফর্ম।
- `#field-1` থেকে `#field-16`: ফর্মের ১৬টি ইনপুট ফিল্ড।
- `#table-main-ledger`: রেজিস্টার পেজের মূল টেবিল।
- `#protocol-card`: ল্যান্ডিং পেজের সিকিউরিটি কার্ড।
- `#hero-section`: ল্যান্ডিং পেজের মেইন ব্যানার।

### ৪. স্টাইল গাইডলাইন:
- বর্তমানের ব্লু-সレート (Blue-Slate) এবং হোয়াইট ক্লিন ডিজাইন হুবহু বজায় রাখতে হবে।
- ফন্ট ফ্যামিলি ('Hind Siliguri') এবং ফন্ট সাইজগুলো পরিবর্তন করা যাবে না।

**এই ফাইলটি পড়া মাত্রই এআই-কে নিশ্চিত করতে হবে যে সে এই প্রোটোকল মেনে কাজ করছে।**
